h1. Specification Summary: okf-schema

*JIRA*: N/A (Open-source project)

h2. What This Change Brings

A new open-source Python package {{okf-schema}} that provides a CLI tool and Python library for working with *OKF (Open Knowledge Format)* bundles. OKF is a markdown-based knowledge format where each concept is a markdown file with YAML frontmatter. The package enables users to initialize, validate, format, search, index, and analyze OKF bundles programmatically.

The project is modeled after {{python-extsemver}} but modernized with {{ruff}}, {{ty}}, {{hatch-vcs}}, and trusted PyPI publishing. It includes a Compendium-style SKILL.md to teach agents how to use the tool.

h2. Advantages

* Clean, modern Python packaging with {{src/}} layout and automatic versioning.
* Comprehensive CLI with 10 subcommands covering the full OKF lifecycle.
* Public Python API for programmatic use.
* High test coverage (95% target) ensures reliability.
* Trusted publishing to PyPI via GitHub Actions OIDC.

h2. Drawbacks / Trade-offs

* Greenfield project requires full scaffolding effort.
* No pre-commit hooks (user preference) — relies on CI for quality gates.

h2. User Journeys (summarized)

* *Initialize a new knowledge base*: A user runs {{okf-schema init kb}} and gets a ready-to-use OKF bundle structure.
* *Validate bundle quality*: A user runs {{okf-schema validate ./kb/bundle}} and receives a detailed report of errors and warnings.
* *Format frontmatter consistently*: A user runs {{okf-schema format ./kb/bundle}} to flatten nested lists while preserving comments.
