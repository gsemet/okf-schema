h1. Specification: okf-schema — Modern Python Package for OKF Bundle Management

*JIRA*: N/A (Open-source project)

h2. Overview

Create a new open-source Python project {{okf-schema}} that provides a CLI tool and Python library for working with *OKF (Open Knowledge Format)* bundles. OKF is a markdown-based knowledge format where each concept is a markdown file with YAML frontmatter. The project enables users to initialize, validate, format, search, and analyze OKF bundles programmatically.

The core validation logic is adapted from existing scripts ({{validate_okf.py}}, {{validate_schema.py}}) found in {{nestor-compendium/skills/Engineering/Knowledge_Graphs/okf/scripts/}}, refactored into a clean, modern, testable Python package. The project is modeled after {{python-extsemver}} (uv + pyproject.toml + just + pytest + Sphinx) but modernized with current best practices including {{ruff}} for linting, {{ty}} for type checking, and {{hatch-vcs}} for automatic versioning from git tags.

The package will be published to *PyPI* under the name {{okf-schema}} with trusted publishing (OIDC) via GitHub Actions.

h2. Functional Requirements

h3. Core Functionality

# *Package Scaffolding*: Modern Python package with {{src/}} layout, {{pyproject.toml}}, {{hatchling}} build backend, and {{hatch-vcs}} for automatic versioning from git tags.

# *CLI with 10 Subcommands*:
** {{init <name>}} — Initialize a new OKF bundle with {{bundle/}} (containing {{index.md}} and {{log.md}}) and {{schema/}} directories.
** {{new --path <root> --name <relative-path> [--type <type>] [--title <title>]}} — Create a new concept file with frontmatter template.
** {{validate <bundle-path> [--schema-db <schema-dir>]}} — Validate bundle structure and frontmatter (errors E1–E6, warnings W1–W6).
** {{format <bundle-path> [--check] [--diff]}} — Format frontmatter: flatten nested lists while preserving comments via {{ruamel.yaml}}.
** {{list <bundle-path>}} — List all concepts (one per line: path, type, title).
** {{show <bundle-path> <concept-path>}} — Display a single concept's frontmatter and body.
** {{index <bundle-path>}} — Auto-regenerate {{index.md}} files for each directory in the bundle.
** {{search <bundle-path> <query>}} — Full-text search across concept frontmatter (title, description, type, tags).
** {{graph <bundle-path>}} — Output concept link graph as ASCII adjacency list.
** {{stats <bundle-path>}} — Bundle statistics with bar charts (by type, tags cloud, links).

# *Python API*: Public Python API exposing core validation ({{validate_bundle}}), formatting ({{format_bundle}}), and utility functions for programmatic use.

# *Built-in Minimal Schema*: A minimal built-in OKF schema validating that frontmatter contains at minimum a {{type}} field (string).

# *Validation Rules*:
* *E1** (Error): Missing or unparseable YAML frontmatter
* *E2** (Error): Missing or empty {{type}} field
* *E3** (Error): Reserved file ({{index.md}}, {{log.md}}) has unexpected structure
* *E4** (Error): Schema validation failure (when schema DB provided)
* *E5** (Error): Unflatten list in frontmatter
* *E6** (Error): Reserved file naming conflict
* *W1** (Warning): Missing recommended field {{title}} or {{description}}
* *W2** (Warning): Broken cross-link
* *W3** (Warning): Missing {{timestamp}} field
* *W4** (Warning): Directory missing {{index.md}}
* *W5** (Warning): {{log.md}} date headings not ISO 8601
* *W6** (Warning): Schema database file not found for declared type

# *Formatter Behavior*: Uses {{ruamel.yaml}} to preserve comments and formatting while flattening nested lists in frontmatter. Supports {{--check}} (exit 1 if changes needed), {{--diff}} (show diff), and default in-place modification.

# *Sphinx Documentation*: API reference via {{autodoc}}, CLI reference via {{sphinx-click}}, using {{furo}} theme and {{myst-parser}} for Markdown support.

# *GitHub Actions CI/CD*:
** {{ci.yml}}: Test matrix (Python 3.10–3.13 on ubuntu-latest; macos-latest and windows-latest for Python 3.12 only). Runs {{just preflight}}.
** {{publish.yml}}: Trusted publishing to PyPI on release via OIDC.

# *In-Project SKILL.md*: Compendium-style skill in {{skills/okf-schema/SKILL.md}} to teach agents how to use {{okf-schema}}.

# *Additional Files*: {{CONTRIBUTING.md}}, {{.editorconfig}}.

h3. Edge Cases

* Bundle path does not exist or is not a directory → exit 2 with clear error message.
* {{init}} target directory already exists → exit 1.
* {{new}} file already exists → exit 1.
* Missing required flags for {{new}} → exit 2 with usage.
* Schema DB directory provided but empty → warnings only (W6).
* External links in markdown → skipped during broken-link check.
* Empty bundle (no markdown files) → valid but warnings about missing index.md.

h2. Non-Functional Requirements

* *Performance*: Validation of a 100-file bundle completes in < 1 second on modern hardware.
* *Security*: No execution of user-provided content; safe YAML parsing via {{ruamel.yaml}}.
* *Compatibility*: Python 3.10, 3.11, 3.12, 3.13.
* *Maintainability*: 95% test coverage minimum; clear separation between CLI ({{click}}) and core logic.
* *Documentation*: Sphinx docs with API and CLI reference; README with quickstart.

h2. Integration Points

* *PyPI*: Published via GitHub Actions trusted publishing (OIDC).
* *GitHub*: Repository at {{https://github.com/gsemet/okf-schema}}.
* *Sphinx*: Documentation built and publishable to ReadTheDocs or GitHub Pages.

h2. Constraints and Assumptions

h3. Constraints

* Must use {{ruamel.yaml>=0.18.0}} for all YAML operations to preserve comments.
* Must use {{src/}} layout.
* Must use {{hatchling}} + {{hatch-vcs}} for build/versioning.
* Must not add interactive prompts to CLI (flags-only for scriptability).
* Must not use pre-commit hooks.

h3. Assumptions

* Users have Python 3.10+ installed.
* OKF bundles follow the documented structure: markdown files with YAML frontmatter.
* Schema files are named {{<type>.schema.json}} or {{<type>.schema.yaml}}.

h2. Out of Scope

* Web UI or server component.
* External database integration.
* Real-time collaboration features.
* Interactive {{new}} command prompts (flags-only).
* Pre-commit hooks.
